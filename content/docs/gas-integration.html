GAS Integration

Bind Gameplay Ability System attributes directly to agent parameters. Prefer attribute handles when available.

Attach ASC (optional but recommended)

ACINAgentBase exposes:

// Blueprint
SetAbilitySystemComponent(ASC)
GetAbilitySystemComponent()

Binding by name

// Blueprint
BindToGameplayAttribute("Health", "Attributes.Health", true, (0, 100))
BindToGameplayAttribute("Stamina", "Attributes.Stamina", true, (0, 100))
BindToGameplayAttribute("Mana", "Attributes.Mana", true, (0, 100))

Binding by handle (preferred)

// Blueprint (assuming you resolved FGameplayAttribute handle)
BindToGameplayAttributeByHandle("Health", HealthAttr, true, (0, 100))

Common patterns

  • Normalize to [0..1] at bind time using the expected max values.
  • Use RefreshParameterBindings() if the ASC, attribute set, or owning actor changes at runtime.
  • Keep game logic outside the plugin; let normalized parameters drive your higher-level BP decision graphs.

Troubleshooting

  • Zero values: Attribute name or handle not found. Validate the set is present and replicated.
  • Jitter: Apply simple smoothing in your BP if attributes update very frequently.
