Learning & Adaptation

Keep it Blueprint-friendly: adapt thresholds and preferences over time, persist between sessions, and remain data-first.

Practical patterns

  • Adaptive thresholds: If an agent frequently reaches low health, gently raise the Retreat sensitivity by storing a small offset and applying it when computing scores.
  • Session persistence: Save per-agent tuning (e.g., curve offsets) to a save game slot or a Data Asset you manage.
  • Contextual tuning: Adjust based on map, time-of-day, or faction using tags you track in your own BP systems.

Workflow

  1. Read normalized parameters each tick or on events.
  2. Record simple counters (times retreated, times flanked, etc.).
  3. Every N seconds, adjust small biases; clamp to sane ranges.
  4. Save periodically and on level transition.
This keeps learning on the project side. The plugin remains generic and unopinionated.
