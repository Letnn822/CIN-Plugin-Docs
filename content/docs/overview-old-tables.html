What is Utility AI?

Utility AI is a decision-making pattern where agents score multiple possible actions based on context, then execute the highest-scoring action. Unlike behavior trees or finite state machines, Utility AI gracefully handles complex situations with many variables by reducing them to simple numeric scores.

How CIN Plugin Works

  1. Services Gather Inputs: Perception, memory, squad data auto-publish to Named Inputs (0-1 normalized values)
  2. Actions Score Themselves: Each action evaluates how suitable it is based on current game state
  3. Anti-Thrashing Systems: Momentum, Resistance, and Locking prevent rapid action switching
  4. Best Action Executes: Highest-scoring action runs (Behavior Tree, Animation, Blueprint logic)
  5. Continuous Re-evaluation: Brain constantly checks if conditions changed enough to warrant switching
  6. Visual Debugging: Overhead widgets show current action, all scores, and input values in real-time

CIN Plugin Features

  • Named Inputs: Normalized 0-1 values representing game state (Health, AmmoCount, EnemyDistance, etc.)
  • Actions: Discrete behaviors (Attack, TakeCover, Reload, Investigate) with scoring curves
  • Services: Automatic input providers that publish perception, memory, and squad data
  • Brain Component: Evaluates actions every tick and executes the best choice
  • Task Momentum: Reduces scores of recently-used actions to prevent repetition
  • Replace Resistance: New actions must score significantly higher to interrupt
  • Task Locking: Failed actions lock temporarily to prevent repeated failures
  • Service Conditions: Services start/stop automatically based on input thresholds

Real Example: Combat Soldier (5 Minutes)

// Services auto-publish these Named Inputs (no setup required):
// - Health (0-1, from GAS or health component)
// - AmmoCount (0-1, normalized to clip size)
// - HasKnownEnemy (0/1, from PerceptionMonitor service)
// - NearestEnemyDistance (0-1, normalized to engagement range)
// - ThreatLevel (0-1, calculated by perception service)
// - InCover (0/1, from environmental query)

// Actions score themselves with simple curves:
Action_Attack:
  Score = HasKnownEnemy * (1 - NearestEnemyDistance) * AmmoCount * 1.5
  // High score when enemy is close and we have ammo

Action_Reload:
  Score = (1 - AmmoCount) * 2.0
  // Urgent when low on ammo

Action_TakeCover:
  Score = ThreatLevel * (1 - Health) * (1 - InCover)
  // Critical when under threat and injured

Action_Retreat:
  Score = (1 - Health) * ThreatLevel * (1 - HasAllyNearby) * 1.8
  // Very high priority when hurt, threatened, and alone

// Brain picks highest score every 0.1s
// Momentum prevents rapid switching
// If action fails (target dies), it locks briefly
// Debug widget shows all scores in real-time overhead

Result: Intelligent combat AI that attacks aggressively when healthy, takes cover when threatened, retreats when overwhelmed, and always reloads at the right time. No complex tree logic needed.

Core Components

  • UUtilityBrainComponent — Main AI brain, lives on AIController, evaluates and executes actions
  • UUtilityActionAsset — Data Asset defining an action with scoring curves and execution logic
  • UUtilityService — Input providers (Perception, Memory, Squad) that publish Named Inputs
  • UCIN_AgentPreset — Data Asset bundling archetype, actions, and services for quick setup
  • UCINQuickSetupComponent — One-click component for applying presets to pawns
  • UCINDebugComponent — Overhead widget showing current action, scores, and inputs

Key Services (Auto-Wired with Archetypes)

  • UService_PerceptionMonitor — Auto-publishes 20+ perception inputs (enemies, allies, threats, memory)
  • UService_EnvironmentalMemory — Tracks object states for investigation behaviors
  • UService_TacticalSense — Reports squad intel to Tactical Squad Component
  • UService_TacticalStrategy — Leader synthesizes squad strategy and publishes to members
  • UService_TagToInput — Maps GameplayTags to Named Inputs for dynamic conditions
