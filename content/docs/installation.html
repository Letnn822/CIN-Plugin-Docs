Installation

Get CIN Plugin up and running in your Unreal Engine 5.5 or 5.6 project. Simple installation with verification steps to ensure everything works correctly.

5 minutes Beginner

Requirements

Before You Begin:
  • Unreal Engine: 5.5 or 5.6
  • Platform: Windows, Mac, or Linux
  • Build Configuration: Development Editor recommended
  • Project Type: C++ or Blueprint (C++ recommended for full features)

Installation Methods

Method 1: Plugin in Project (Recommended)

  1. Copy Plugin to Project:

    YourProject/
└── Plugins/
    └── CINPlugin/
        ├── CINPlugin.uplugin
        ├── Source/
        │   ├── CINCore/
        │   ├── CINAI/
        │   ├── CINPerception/
        │   ├── CINLOD/
        │   └── CINEditor/
        └── Resources/

  2. Regenerate Project Files:

    • Right-click YourProject.uproject → Generate Visual Studio project files
    • Or use command line: UnrealBuildTool -projectfiles -project="YourProject.uproject"

  3. Build the Project:

    • Open YourProject.sln in Visual Studio / Rider
    • Build configuration: Development Editor
    • Platform: Win64 (or your target platform)
    • Build solution

  4. Enable Plugin in Editor:

    • Open your project in Unreal Editor
    • Edit → Plugins
    • Search for "CIN" or navigate to Project → Other
    • Enable CINPlugin
    • Restart editor when prompted

Method 2: Engine Plugins (Advanced)

For shared access across multiple projects:

UnrealEngine/
└── Engine/
    └── Plugins/
        └── CINPlugin/

Same steps as Method 1, but plugin will be available to all projects using that engine installation.

Engine Plugins Note: Updating the plugin requires restart of all projects using it. Method 1 (project plugins) recommended for active development.

Verification Steps

Step 1: Check Plugin Modules

  1. Edit → Plugins → Search "CIN"
  2. Verify CINPlugin shows as Enabled
  3. Click on plugin to see details
  4. Verify modules listed:
    • CINCore - Core types and interfaces
    • CINAI - Utility AI and actions
    • CINPerception - Perception integration
    • CINLOD - Performance management
    • CINEditor - Editor tools (Editor only)

Step 2: Verify Classes Available

Test that plugin classes are accessible:

  1. Create AI Controller Blueprint:

    • Content Browser → Right-click → Blueprint Class
    • Parent Class: AIController
    • Name: BP_TestAIController

  2. Add CIN Components:

    Open BP_TestAIController, click Add Component, verify these are available:

    • UtilityBrainComponent (CINAI module)
    • CINQuickSetupComponent (CINAI module)
    • CINDebugComponent (CINAI module)

    If you see these components, installation is successful!

Step 3: Create Test Data Asset

  1. Content Browser → Right-click → Miscellaneous → Data Asset
  2. Search for: CIN_AgentPreset
  3. If this appears, CIN classes are properly registered ✅

Step 4: Check Editor Tools

  1. Top menu → Tools
  2. Look for CIN submenu
  3. Verify tools available:
    • Create Demo AI Trees + Blackboard
    • Build Demo Hub
Installation Complete! If all verification steps passed, CIN Plugin is ready to use.

Troubleshooting

"Plugin failed to load"

Symptoms: Error on startup, plugin shows as disabled

Solutions:

  1. Verify project is C++ project (or convert to C++):
    • File → New C++ Class → Create any class to convert
  2. Regenerate project files
  3. Clean and rebuild:
    • Delete Binaries/, Intermediate/, Saved/
    • Regenerate and rebuild
  4. Check Output Log for specific error messages

"Missing modules" error

Symptoms: CINCore, CINAI modules not found

Solutions:

  1. Verify plugin structure matches installation layout
  2. Check CINPlugin.uplugin exists and is valid JSON
  3. Ensure all module folders exist: 
    Plugins/CINPlugin/Source/
├── CINCore/
├── CINAI/
├── CINPerception/
├── CINLOD/
└── CINEditor/
  4. Each should have [ModuleName].Build.cs file

"Classes not appearing in editor"

Symptoms: Can't find UtilityBrainComponent, CIN_AgentPreset, etc.

Solutions:

  1. Restart editor after enabling plugin
  2. Verify plugin is enabled (Edit → Plugins)
  3. Check if build succeeded without errors
  4. Try closing editor, deleting Saved/ folder, reopening

"Compile errors"

Common Issues:

Error Cause Solution
Cannot open include file Missing engine modules Verify UE 5.5 or 5.6 installed
Unresolved external symbol Module dependency issue Regenerate project files
Incompatible engine version Wrong UE version Update to UE 5.5+ or 5.6

Module Overview

Understanding what each module provides:

Module Purpose Key Classes
CINCore Foundation types, interfaces, managers ICINContext, Core managers
CINAI Utility AI, actions, services UUtilityBrainComponent, UUtilityActionAsset, UUtilityService
CINPerception Perception integration UService_PerceptionMonitor, Advanced perception
CINLOD Performance & LOD management UCINLODManager, Budget systems
CINEditor Editor tools & automation Demo builders, asset tools

  • Editor tools available

  • Debug tools compiled out
  • Minimal overhead
  • Use for final builds
