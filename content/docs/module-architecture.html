Module Architecture

Technical overview of CIN Plugin's module structure, dependencies, and extensibility points for advanced developers.

12 minutes Reference

Module Structure

CINPlugin/
├─ CINAI (Runtime Module)
│  ├─ Public/
│  │  ├─ Components/
│  │  ├─ Config/
│  │  ├─ Services/
│  │  ├─ Executors/
│  │  ├─ Providers/
│  │  └─ Blueprint/
│  └─ Private/
│     └─ Implementation files
└─ CINPlugin.uplugin (Plugin descriptor)

Core Classes

Brain System

  • UUtilityBrainComponent - Main brain component
  • UUtilityActionAsset - Action data assets
  • UUtilityBrainPreset - Brain configuration

Execution

  • UUtilityActionExecutor - Base executor
  • UUtilityActionExecutor_BehaviorTree - BT executor
  • UUtilityActionExecutor_Blueprint - BP executor

Services

  • UUtilityService - Base service class
  • UService_PerceptionMonitor - Perception integration
  • UService_TacticalSense - Squad perception
  • UService_TacticalStrategy - Squad strategy

Dependencies

// CINAI.Build.cs
PublicDependencyModuleNames.AddRange(new string[] 
{
    "Core",
    "CoreUObject",
    "Engine",

});

PrivateDependencyModuleNames.AddRange(new string[] 
{
    "UMG",
    "Slate",
    "SlateCore"
});

Extension Points

Custom Services

Inherit from: UUtilityService
Override: TickService(), OnServiceStart(), OnServiceStop()
Location: YourGame/Source/YourGame/AI/

Custom Executors

Inherit from: UUtilityActionExecutor
Override: Execute(), TickExecute(), StopExecution()
Location: YourGame/Source/YourGame/AI/Executors/

Custom Components

Extend: UActorComponent
Integration: Publish to NamedInputs via brain
Location: YourGame/Source/YourGame/AI/Components/

Build Configuration

// YourGame.Build.cs
PublicDependencyModuleNames.AddRange(new string[] 
{
    "CINAI" // Add CIN Plugin dependency
});

// Now you can #include CIN headers:
#include "UtilityBrainComponent.h"
#include "UtilityService.h"
