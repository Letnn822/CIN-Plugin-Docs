Welcome to CIN Plugin

CIN (Contextual Intelligence Network) is a professional Utility AI framework for Unreal Engine 5.5+. Built for AAA-quality game AI with Blueprint and C++ support, competitive features, and comprehensive debugging tools.

What is Utility AI?

Utility AI is a decision-making pattern where agents score multiple possible actions based on context, then execute the highest-scoring action. Unlike behavior trees or finite state machines, Utility AI gracefully handles complex situations with many variables by reducing them to simple numeric scores.

CIN Plugin implements Utility AI with:

  • Named Inputs: Normalized 0-1 values representing game state (Health, AmmoCount, EnemyDistance, etc.)
  • Actions: Discrete behaviors (Attack, TakeCover, Reload, Investigate) with scoring curves
  • Services: Automatic input providers that publish perception, memory, and squad data
  • Brain Component: Evaluates actions every tick and executes the best choice

How CIN Works

  1. Services Gather Inputs: Perception, memory, squad data auto-publish to Named Inputs
  2. Brain Evaluates Actions: Every action scores itself based on current inputs
  3. Momentum & Locking: Prevents action thrashing with score penalties and temporary locks
  4. Best Action Executes: Highest-scoring action runs (Behavior Tree, Animation, Blueprint logic)
  5. Continuous Re-evaluation: Brain constantly checks if a better action is available
  6. Debug Visualization: Overhead widgets show current action, scores, and inputs in real-time

5-Minute Quick Start

  1. Add UUtilityBrainComponent to your AI Controller
  2. Create a UCIN_AgentPreset Data Asset and assign an archetype (BaseCombat, BaseStealth, etc.)
  3. Create UUtilityActionAsset data assets for your AI's actions (Attack, Patrol, Investigate)
  4. Use UCINQuickSetupComponent for one-click initialization, or call ApplyAgentPresetToPawn
  5. Press Play—the Brain evaluates actions and debug widgets show the decision process

Key Features

Core Utility AI

  • Dynamic Action Scoring: Actions score themselves based on Named Inputs (0-1 normalized)
  • Task Momentum: Reduces scores of recently-used actions to prevent repetition
  • Replace Resistance: New actions must score significantly higher to interrupt current action
  • Task Locking: Failed actions lock temporarily, preventing repeated failures
  • Service Conditions: Services start/stop automatically based on input thresholds

Advanced Perception

  • Stimulus Memory: Tracks last known positions and visibility status of enemies
  • Threat Assessment: Automatic 0-1 threat levels based on distance, visibility, and recency
  • 20+ Auto-Published Inputs: EnemyCount, NearestEnemyDistance, ThreatLevel, TimeSinceLastSawEnemy, etc.
  • Team Affiliation: Uses GenericTeamId to distinguish allies from enemies

Squad Coordination

  • Tactical Squad Component: Agents auto-join squads by name and elect leaders
  • Shared Strategy: Leader synthesizes squad strategy (Search/Assault/Flank/Hold/Retreat)
  • Coordinated Inputs: Squad data published as NamedInputs (IsLeader, SquadStrategy, MemberIndex)
  • Deterministic Roles: Squad members get consistent roles for flanking maneuvers

Environmental Memory

  • Object State Tracking: Remembers doors, containers, displaced objects
  • Investigation Actions: Pre-built actions for investigating suspicious changes
  • Subsystem Integration: Global tracking of all memorizable objects in the level
  • GameplayTag Integration: Uses tags for object classification and state changes

Core Components

  • UUtilityBrainComponent — Main AI brain, lives on AIController, evaluates and executes actions
  • UUtilityActionAsset — Data Asset defining an action with scoring curves and execution logic
  • UUtilityService — Input providers (Perception, Memory, Squad) that publish Named Inputs
  • UCIN_AgentPreset — Data Asset bundling archetype, actions, and services for quick setup
  • UCINQuickSetupComponent — One-click component for applying presets to pawns
  • UCINDebugComponent — Overhead widget showing current action, scores, and inputs

Service Classes

  • UService_PerceptionMonitor — Auto-publishes 20+ perception inputs (enemies, allies, threats)
  • UService_EnvironmentalMemory — Tracks object states for investigation behaviors
  • UService_TacticalSense — Reports squad intel to Tactical Squad Component
  • UService_TacticalStrategy — Leader synthesizes squad strategy and publishes to members
  • UService_TagToInput — Maps GameplayTags to Named Inputs for dynamic conditions

Blueprint Libraries

  • UCIN_UtilityAILibrary — Query/modify actions, momentum, locking, debug tools
  • UCIN_AIBlueprintLibrary — Apply presets, configure agents, manage services
  • UCIN_AdvancedPerceptionLibrary — Query perception data, stimulus memory, threats

Real Example: Combat Soldier

// Services auto-publish these Named Inputs:
// - Health (0-1, from GAS or health component)
// - AmmoCount (0-1, normalized to clip size)
// - HasKnownEnemy (0/1, from perception)
// - NearestEnemyDistance (0-1, normalized to engagement range)
// - ThreatLevel (0-1, calculated by perception service)

// Actions score themselves:
Action_Attack: Score = HasKnownEnemy * (1 - NearestEnemyDistance) * AmmoCount
Action_Reload: Score = (1 - AmmoCount) * 2.0  // High priority when low ammo
Action_TakeCover: Score = ThreatLevel * (1 - Health)
Action_Retreat: Score = (1 - Health) * ThreatLevel * (1 - HasAllyNearby)

// Brain picks highest score, momentum prevents rapid switching
// If Attack fails (target dies), it locks briefly so agent considers other options
