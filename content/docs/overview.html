Welcome to the CIN Plugin

CIN is a designer-friendly AI framework for Unreal Engine 5.6+. It focuses on normalized, data-driven parameters and modular systems that you can wire into existing gameplay without hardcoding logic.

What is CIN? (Beginner‑friendly)

Think of CIN as a set of simple knobs (numbers from 0 to 1) that describe your game at any moment: Health, Stamina, DistanceToPlayer, Alertness, etc. CIN reads those knobs and helps your AI make smart choices without you writing custom C++ for each case. You connect your existing systems to CIN with a few Blueprint nodes.

How it works in simple terms

  1. You expose game signals (like Health) to CIN using Parameter Binding.
  2. The Brain gathers signals and context (world, self, social).
  3. Intelligence layers each give an opinion (reactive, utility, predictive, social, learning).
  4. Arbitration Strategy mixes those opinions into one decision.
  5. Your game executes the chosen action however you like (C++, Blueprint, Ability System).
  6. Learning can adjust priorities over time (optional).

5‑minute Quick Start

  1. Create or use an agent based on ACINAgentBase (your AI pawn/character).
  2. In its Blueprint, bind one parameter (e.g., Health) using the nodes from Parameter Binding. Start with a simple Direct Value if needed.
  3. Pick defaults: a Role Archetype (personality) and an Arbitration Strategy (how layers are mixed). Optional: add Environment Semantics.
  4. Press Play. As Health or Distance changes, the Brain updates decisions smoothly.
ℹ️
Where does gameplay live? CIN is generic AI logic. Any combat, abilities, or animations are yours. Our sample combat wrappers live in your project at Source/CIN/Combat/ and just show how to bind external systems using the same parameter binding API.

What you can do

  • Drive AI with real game signals using parameter bindings. Examples: Health, Stamina, Alertness, DistanceToPlayer.
  • Integrate with GAS in a few Blueprint calls via ACINAgentBase binding utilities.
  • Perceive the world with UCINPerceptionComponent and react to stimuli.
  • Scale performance with Intelligence LOD controls and context-aware updates.
  • Keep your project code separate: the plugin ships generic AI; project modules can add game-specific wrappers (e.g., combat helpers).

Core building blocks

  • ACINAgentBase — Your AI pawn base. Exposes Blueprint-callable functions for parameter binding and state.
  • FCINParameterBinding, FCINParameterBindingSet, ECINParameterBindingType — The parameter system.
  • UCINPerceptionComponent — Unified perception with configurable senses and events.
  • UCINBrain — Internal brain class that maintains context and supports goal-driven behavior.
  • Action Packs — Data assets listing available actions with animations and requirements.
  • Goal Templates — Data assets for runtime goal generation, priorities, and completion rules.
ℹ️
Project-side combat helpers: The UCINCombatBlueprintLibrary and related combat components are in your project module under Source/CIN/Combat/, not inside the plugin. They demonstrate how to integrate external systems using the parameter bindings.

Real-life example

Bind your agent's Health, Stamina, and DistanceToPlayer as normalized parameters (0–1). Use them to switch goals between "Advance", "Retreat", and "CallForHelp". As the player approaches or your health drops, the agent updates behavior smoothly without any hardwired gameplay logic.
