Glossary of Terms

Complete reference of terminology used in CIN Plugin and Utility AI systems.

10 minutes Reference

Core Concepts

Action

A behavior the AI can choose to perform. Contains considerations for scoring and an executor for implementation.

Archetype

Pre-configured AI template providing foundational behaviors (BaseCombat, BaseStealth, etc.).

Base Score

Starting score value for an action before considerations are applied (typically 30-90).

Brain

The UUtilityBrainComponent that manages action evaluation and execution.

Consideration

A single factor in action scoring. Maps an input value through a response curve to affect final score.

Executor

Component that implements action behavior (Behavior Tree, Blueprint, or custom C++).

Input Provider

Component that publishes data to Named Inputs for use in action scoring.

Momentum

Penalty applied to recently-executed actions to prevent repetitive behavior.

Named Input

Key-value pair (FName â†’ float) representing world state, published by services.

Preset

Configuration asset containing actions, services, and settings for an AI type.

Priority

Tie-breaker value when actions have equal scores (0-100).

Replace Resistance

Multiplier requiring new action to score higher to interrupt current action (reduces thrashing).

Response Curve

Curve mapping input value (0-1) to score contribution (0-100) in a consideration.

Service

Component that runs periodically to gather data and publish Named Inputs.

Squad

Group of AI agents coordinating via TacticalSquadComponent and related services.

Utility AI

AI architecture using score-based decision making instead of priority or rule-based systems.

Advanced Terms

Task Locking

Preventing an action from being selected for a duration after failure or abortion.

Score Modifier

Runtime multiplier applied to action score for dynamic behavior adjustment.

Stimulus Memory

Remembered perceptions (enemy last-known positions) maintained by PerceptionMonitor service.

Threat Level

Calculated danger assessment (0-1) based on enemy distance, count, and visibility.

Tactical Strategy

Squad-level decision (Search/Assault/Flank/Hold/Retreat) synthesized by leader.
